# Homework Project: Implementing GitOps with Gitea and ArgoCD in Kubernetes

## Objective

The goal of this project is to introduce students to GitOps principles by setting up a continuous deployment pipeline using Gitea and ArgoCD within a Kubernetes cluster. Students will:

- Install and configure Gitea as a self-hosted Git service.
- Install and configure ArgoCD to manage application deployments from Gitea repositories.
- Deploy a web application using ArgoCD.
- Implement a liveness probe for the Nginx application to ensure its health and availability.
- Package their builds for easy redeployment.

---

## Task 0: Setup (0 points)

### Environment Preparation

We will utilize a virtual machine provided by the [faculty's cloud](http://cloud.grid.pub.ro/) infrastructure.

:::warning
All virtual machines **must** be created under the `didactic-cc-homework` OpenStack project.

Failing to do so may result in your VM being deleted.
:::

**Virtual Machine Configuration:**

- **Instance Name:** `HW-<Your LDAP username>` (e.g., `HW-john.doe`)
- **Instance Boot Source:** Boot from image
- **Image Name:** CC Template
- **Flavor:** `g.medium`

**Initial Setup:**

1. Access the virtual machine via SSH.
2. Update system packages.
3. Check if essential tools are installed (Git, curl, wget, vim).
4. Check if Docker is installed.
5. Check if `kubectl` is installed.
6. Check if [Kind (Kubernetes in Docker)](https://kind.sigs.k8s.io/) is installed.
7. Check if Helm is installed.

## Task 1: Create a Kubernetes Cluster (5 points)

1. Initialize a Kubernetes cluster using Kind. (1 point)
2. Configure `kubectl` to use the new cluster. (2 points)
3. Verify the cluster status and ensure nodes are ready. (2 points)

---

## Task 2: Install [Gitea](https://docs.gitea.com/) (20 points)

1. Create a namespace for Gitea. (2 points)
2. Add the Gitea Helm repository. (2 points)
3. Install Gitea using Helm. (2 points)
4. Expose Gitea service to access the web interface. Hint: proxy to your browser. (10 points)
5. Complete the initial Gitea setup:
   - Register a new user. (2 points)
   - Create a repository named `nginx-deployment`. (2 points)

---

## Task 3: Install [ArgoCD](https://argo-cd.readthedocs.io/en/stable/) (20 points)

1. Create a namespace for ArgoCD. (2 points)
2. Install ArgoCD. (2 points)
3. Expose the ArgoCD server. Hint: proxy to your browser. (3 points)
4. Log in to ArgoCD. (3 points)
5. Connect the Gitea repository to ArgoCD. (10 points)

---

## Task 4: Deploy Nginx with ArgoCD (30 points)
For this task, you are going to deploy your own [cloud-courses](https://gitlab.cs.pub.ro/SCGC/cloud-courses) page.

1. Create a Kubernetes manifest for an [Nginx Reverse Proxy](https://docs.nginx.com/nginx/admin-guide/web-server/reverse-proxy/) deployment and service to host your own [cloud-courses](https://gitlab.cs.pub.ro/SCGC/cloud-courses) page in your Gitea repository. (10 points)
2. Push the manifest to the Gitea repository. (2 points)
3. Create an ArgoCD application that points to the repository and deploys the manifest. (8 points)
4. Apply the ArgoCD application manifest. (5 points)
5. Verify that the Nginx deployment and service are running. (5 points)

---

## Task 5: Add Liveness Probe to Nginx (10 points)

1. Modify the deployment manifest to include a liveness probe. (2 points)
2. Push the updated manifest to Gitea. (1 points)
3. Confirm that ArgoCD syncs the changes and redeploys the application. (3 points)
4. Verify that the liveness probe is configured correctly. (4 points)

---

## Task 6: Package the Nginx with Helm (20 points)

1. Create a Helm chart that deploys the previously configured Nginx deployment and service. (10 points)
2. Publish the package on your Gitea repository. Hint: use the [package registry](https://docs.gitea.com/usage/packages/overview/). (5 points)
3. Verify that your Helm chart deploys the previously configured Nginx deployment and service. (5 points)
